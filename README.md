# 🧠 Offline GPT-2 Portable Assistant

**Offline GPT-2 Portable Assistant** — это полностью автономная языковая модель, упакованная в портативный исполняемый файл. Проект позволяет запускать нейросетевой текстовый генератор на любом компьютере с ОС Windows без необходимости установки Python, библиотек или наличия интернет-соединения.

---

## 🚀 Ключевые особенности

* **Zero-Cloud Architecture:** Все вычисления происходят локально на CPU. Конфиденциальность данных гарантирована на аппаратном уровне.
* **Портативность:** Проект оптимизирован для работы с внешних USB-накопителей.
* **Safetensors Technology:** Использование современного формата весов обеспечивает безопасность и ускоренную загрузку через механизм `memory mapping (mmap)`.
* **CI/CD Pipeline:** Автоматизированная сборка через GitHub Actions исключает конфликты зависимостей и гарантирует воспроизводимость среды.

## 🧬 Техническая реализация (Under the Hood)

Проект использует архитектуру **Transformer**. В процессе инференса модель оперирует механизмом самовнимания (Self-Attention), который вычисляется по формуле:

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

### Алгоритм работы:

1. **Инициализация:** Загрузка весов `model.safetensors` в RAM через `mmap`.
2. **Токенизация:** Преобразование входной строки в числовые идентификаторы (BPE токенизация).
3. **Генерация:** Предсказание следующего токена на основе контекстного окна в 1024 токена.
4. **Декодирование:** Обратное преобразование векторов в человекочитаемый текст.

## 📦 Системные требования и установка

### Требования:

* **ОЗУ:** 4 ГБ (свободных не менее 1.5 ГБ).
* **Место на диске:** ~1.4 ГБ (включая зависимости и веса).
* **Процессор:** Поддержка инструкций AVX2 (для ускорения вычислений).

### Инструкция по развертыванию:

1. Скачайте актуальную сборку из раздела **Actions** или **Releases**.
2. Распакуйте архив в целевую директорию на жестком диске или флеш-карте.
3. Убедитесь, что папка `model/` с весами находится в корне проекта.
4. Запустите `GPT2_Offline.exe`.

## 🏗 CI/CD Конвейер

Сборка исполняемого файла автоматизирована с помощью **GitHub Actions**. Это позволяет получать стабильный билд после каждого изменения кода:

* Инициализация виртуальной машины `windows-latest`.
* Установка легковесной версии `torch-cpu` для минимизации веса дистрибутива.
* Сборка через PyInstaller с флагом `--collect-all`, что гарантирует перенос всех метаданных библиотек `transformers` и `accelerate`.

## ⚠️ Дисклеймер

Данный проект является **продолжателем текста**, а не чат-ботом. Модель обучалась на англоязычном сегменте интернета, поэтому наилучшие результаты достигаются при вводе запросов на английском языке.

> **Note:** При первом запуске возможна задержка в 10-30 секунд — это время требуется для развертывания графа вычислений в оперативной памяти.
